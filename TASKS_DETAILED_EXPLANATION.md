## 📋 任务详解

### 🔴 **Task 1: OLED 命令解析与电流控制 (TASK_OLED_CTRL)**

**职责**：
```
接收 UART 命令 → 解析电流值 → 设置 DAC → 反馈结果
```

**具体工作**：

| 项目 | 详情 |
|------|------|
| **输入** | UART 串口接收到的命令字符串（如："set_curr(1500.5mA)"） |
| **处理** | 1. 字符串解析：提取整数部分和小数部分 |
|        | 2. 范围检查：验证电流值是否在 0~13000mA 之间 |
|        | 3. 设备检查：确认 AD5667 DAC 设备就绪 |
| **输出** | 1. 向 DAC 模块写入电流设置值 |
|        | 2. 打印日志反馈（成功/失败） |
| **优先级** | Normal (2) |
| **堆栈** | 512 字节 |
| **执行周期** | ~100ms |

**代码流程示例**：
```c
// 接收命令："set_curr(1500.5mA)"
Task_ParseCurrentValue(buffer)
  ↓
// 返回：1500.5
  ↓
Task_ApplyCurrentSetting(1500.5)
  ↓
// 检查 AD5667 就绪？
if (AD5667_IsDeviceReady() == HAL_OK)
  ↓
// 设置电源和电流值
AD5667_Writepower(1, 0)     // 使能电源
AD5667_WritData_setcurr(1500) // 设置电流
  ↓
// 更新全局变量
g_current_set = 1500.5
  ↓
// 日志输出
LOG_INFO("Current set to: 1500.5 mA")
```

**实际应用场景**：
- 用户通过串口工具发送："set_curr(5000mA)"
- 电子负载开始以 5000mA 的电流工作
- 设置成功日志输出到控制台

### 🟢 **Task 2: LCD 显示更新 (TASK_DISPLAY_UPDATE)**

**职责**：
```
定期读取数据 → 格式化显示 → 更新 LCD 屏幕
```

**具体工作**：

| 项目 | 详情 |
|------|------|
| **输入** | 全局变量中的测量数据 |
|        | - `g_voltage`: 电压值 (V) |
|        | - `g_current`: 电流值 (A) |
|        | - `g_temperature`: 温度值 (°C) |
|        | - `g_current_disp`: 设置电流值 (mA) |
| **处理** | 1. 数据分解：整数部分和小数部分 |
|        | 2. 格式化：转换为显示格式 |
|        | 3. 屏幕更新：调用 LCD_ShowxNum() 更新各区域 |
| **输出** | LCD 屏幕显示 |
| **优先级** | Normal (2) |
| **堆栈** | 512 字节 |
| **执行周期** | ~500ms |
| **显示区域** | |
|        | 第 1 行：电压 (V) - 位置 (105, 0) |
|        | 第 2 行：电流 (A) - 位置 (105, 55) |
|        | 第 3 行：功率 (W) - 位置 (90, 110) |
|        | 第 4 行：设置电流 (mA) - 位置 (120, 165) |

**LCD 屏幕布局**：
```
┌──────────────────────────────────┐
│ 电压: 12.345V                    │ (Y=0)
├──────────────────────────────────┤
│ 电流: 2.500A                     │ (Y=55)
├──────────────────────────────────┤
│ 功率: 30.625W                    │ (Y=110)
├──────────────────────────────────┤
│ 设置: 5000.0 mA                  │ (Y=165)
├──────────────────────────────────┤
│ 步进: 100 mA                     │ (Y=200)
└──────────────────────────────────┘
```

**代码流程示例**：
```c
void Task_DisplayUpdate(void *argument)
{
    for (;;) {
        // 读取全局数据
        float voltage = g_voltage;      // 12.345
        float current = g_current;      // 2.500
        
        // 分解数据：整数.小数
        uint16_t v_int = (uint16_t)voltage;     // 12
        float v_dec = voltage - v_int;           // 0.345
        uint16_t v_dec_disp = v_dec * 1000;     // 345
        
        // 显示电压
        LCD_ShowxNum(105, 0, v_int, 2, 32, 0);   // "12"
        LCD_ShowxNum(150, 0, v_dec_disp, 3, 32, 1); // "345"
        
        // 类似处理电流、功率等...
        
        osDelay(500);
    }
}
```

**实际应用场景**：
- 实时显示负载的电压、电流、功率
- 用户可以在屏幕上看到当前设置和测量值
- 每 500ms 刷新一次，响应用户操作

### 🔵 **Task 3: 按键处理 (TASK_KEYPAD_PROCESS)**

**职责**：
```
检测按键按下 → 去抖 → 执行对应操作 → 更新显示
```

**具体工作**：

| 项目 | 详情 |
|------|------|
| **输入** | GPIO 按键信号 (SET, UP, DOWN, CONFIG) |
| **处理** | 1. 去抖延时 (10ms) |
|        | 2. 检查按键是否真正按下 |
|        | 3. 等待按键释放 |
|        | 4. 执行对应的处理函数 |
| **输出** | 电流/模式/步进值改变 |
| **优先级** | Normal (2) |
| **堆栈** | 1024 字节 |
| **执行周期** | ~50ms |

**4个按键说明**：

#### 🔘 **SET 键** - 模式切换
```
功能：切换输出模式 (OFF → + → - → OFF)

按键状态状态转移：
    ┌────────┐
    │   OFF  │ (cc_state=0)
    │ 显示   │
    │  "OFF" │
    └───┬────┘
        │ 按下 SET
        ▼
    ┌────────┐
    │   +    │ (cc_state=1)
    │ 增加   │
    │ 显示   │
    │  "+"   │
    └───┬────┘
        │ 按下 SET
        ▼
    ┌────────┐
    │   -    │ (cc_state=2)
    │ 减少   │
    │ 显示   │
    │  "-"   │
    └───┬────┘
        │ 按下 SET
        ▼
    ┌────────┐
    │  OFF   │
    └────────┘
```

#### ⬆️ **UP 键** - 增加电流
```
功能：按配置的步进值增加电流

流程：
1. 检测按键按下 → 去抖(10ms) → 确认按下
2. 等待按键释放
3. 调用 Task_HandleKeyUp(step)
   - 检查目标电流是否超过最大值 (13000mA)
   - 如果超过，重置为 0 (溢出保护)
   - 否则，通过 Task_ApplyCurrentSetting() 设置新值
4. 更新显示

示例：
  当前电流: 1000mA, 步进值: 100mA
  按下 UP 键 → 电流变为 1100mA
  按下 UP 键 → 电流变为 1200mA
  ...
  按下 UP 键 (当前 13000mA) → 电流重置为 0mA (溢出保护)
```

#### ⬇️ **DOWN 键** - 减少电流
```
功能：按配置的步进值减少电流

流程：
1. 检测按键按下 → 去抖(10ms) → 确认按下
2. 等待按键释放
3. 调用 Task_HandleKeyDown(step)
   - 如果当前电流 >= 步进值，则减少
   - 否则设置为 0
   - 通过 Task_ApplyCurrentSetting() 设置新值
4. 更新显示

示例：
  当前电流: 1000mA, 步进值: 100mA
  按下 DOWN 键 → 电流变为 900mA
  按下 DOWN 键 → 电流变为 800mA
  ...
  按下 DOWN 键 (当前 50mA) → 电流变为 0mA (下限)
```

#### ⚙️ **CONFIG 键** - 改变步进值
```
功能：改变UP/DOWN键的步进大小

流程：
1. 检测按键按下 → 去抖(10ms) → 确认按下
2. 等待按键释放
3. 调用 Task_HandleKeyConfig()
   - 循环切换：1mA → 100mA → 1000mA → 1mA
   - 更新显示，显示当前步进值
4. 后续 UP/DOWN 按键都以新的步进值工作

步进值循环：
  config_state=0: 步进值 = 1mA   显示"1"
  ↓ 按 CONFIG
  config_state=1: 步进值 = 100mA 显示"100"
  ↓ 按 CONFIG
  config_state=2: 步进值 = 1000mA 显示"1000"
  ↓ 按 CONFIG
  config_state=0: 步进值 = 1mA   显示"1"
  ...
```

**按键处理状态机**：
```
                ┌─────────────────────────────┐
                │   等待按键                  │
                │ (key_xxx_pressed = 0)       │
                └────────────┬────────────────┘
                             │ 检测到按键
                             ▼
                ┌─────────────────────────────┐
                │   记录按键按下              │
                │ (key_xxx_pressed = 1)       │
                │ 去抖延时 10ms               │
                └────────────┬────────────────┘
                             │ 确认按键仍按下
                             ▼
                ┌─────────────────────────────┐
                │   等待按键释放              │
                │ (持续循环)                  │
                └────────────┬────────────────┘
                             │ 按键释放
                             ▼
                ┌─────────────────────────────┐
                │   执行处理函数              │
                │ Task_HandleKeyXxx()         │
                │ 更新设置/显示               │
                └────────────┬────────────────┘
                             │
                             ▼
                ┌─────────────────────────────┐
                │   重置按键标志              │
                │ (key_xxx_pressed = 0)       │
                └─────────────────────────────┘
```

### 🟡 **Task 4: 数据采集 (TASK_DATA_ACQUISITION)**

**职责**：
```
周期性读取 ADC → 转换数据 → 计算功率 → 更新全局变量
```

**具体工作**：

| 项目 | 详情 |
|------|------|
| **输入** | ADC/MCP3421 传感器原始数据 |
| **处理** | 1. 触发 ADC 采样 (DMA模式) |
|        | 2. 读取电压值，乘以转换系数 (×3) |
|        | 3. 读取电流值，乘以转换系数 (×2/10/0.05) |
|        | 4. 计算功率: P = V × I |
| **输出** | 更新全局变量 |
|        | - `g_voltage`: 电压 (V) |
|        | - `g_current`: 电流 (A) |
| **优先级** | Normal (1) |
| **堆栈** | 1024 字节 |
| **执行周期** | ~100ms |

**硬件连接与数据流**：
```
┌─────────────────────────────────────────────────────┐
│               被测设备 (Load)                        │
│         ┌──────────┐                                │
│         │ +V  0V ──┼─┐                             │
│         └──────────┘ │                             │
│                      │                             │
│                      ▼                             │
│         ┌─────────────────────────┐                │
│         │   电子负载电路          │                │
│         │ ┌─────────────────────┐ │                │
│         │ │ 电压采样: ADC CH0   │ │ ──► DAC输入  │
│         │ │ 电流采样: ADC CH1   │ │ ──► DAC输入  │
│         │ └─────────────────────┘ │                │
│         └──────────┬──────────────┘                │
│                    │                               │
│         ┌──────────▼──────────┐                    │
│         │  DMA 自动采样       │                    │
│         │ (ADC_DMA 模块)      │                    │
│         └──────────┬──────────┘                    │
│                    │                               │
│         ┌──────────▼──────────┐                    │
│         │  Task_ReadVoltage() │ ──► g_voltage    │
│         │  Task_ReadCurrent() │ ──► g_current    │
│         │  计算功率 P=V×I     │ ──► 用于显示     │
│         └─────────────────────┘                    │
└─────────────────────────────────────────────────────┘

数据转换公式：
  原始值(ADC) ──× 转换系数──► 实际值
  
  电压：ADC_Val × 3                     = V (Volts)
  电流：ADC_Val × 2 / 10 / 0.05         = A (Amperes)
  功率：Voltage × Current               = W (Watts)

示例：
  ADC 电压值: 2048 ──× 3──► 6144 ──÷ 1024──► 6V
  ADC 电流值: 1024 ──× 2/10/0.05──► 40.96A
  功率 = 6V × 40.96A = 245.76W
```

**代码流程示例**：
```c
void Task_DataAcquisition(void *argument)
{
    for (;;) {
        // 1. 采样 ADC (由硬件 DMA 自动完成)
        Task_SampleADC();  // ADC_DMA 自动采样
        
        // 2. 读取转换后的值
        float voltage = DVM_GET(1) * 3;      // 转换系数
        float current = DVM_GET(2) * 2 / 10 / 0.05;
        
        // 3. 更新全局变量 (供其他任务读取)
        g_voltage = voltage;
        g_current = current;
        
        // 示例输出：
        // 采样电压: 12.5V
        // 采样电流: 2.3A
        // 采样功率: 28.75W
        
        osDelay(100);  // 每 100ms 采样一次
    }
}
```

**实际应用场景**：
- 被测设备连接到电子负载
- 每 100ms 自动采集一次电压和电流
- 计算消耗的功率
- 数据用于实时显示和温度监控

### 🔴 **Task 5: 温度监控与保护 (TASK_MONITOR_PROTECTION)**

**职责**：
```
周期性读取温度 → 对比阈值 → 触发保护机制
```

**具体工作**：

| 项目 | 详情 |
|------|------|
| **输入** | 温度传感器数据 (ADC CH2) |
| **处理** | 1. 读取芯片温度 |
|        | 2. 与保护阈值 (70°C) 对比 |
|        | 3. 与警告阈值 (60°C) 对比 |
|        | 4. 采取相应保护措施 |
| **输出** | 1. 记录温度日志 |
|        | 2. 关闭输出或降低电流 |
| **优先级** | Idle (0) - 最低优先级 |
| **堆栈** | 512 字节 |
| **执行周期** | ~1000ms (1秒采样一次) |

**温度保护机制**：
```
┌──────────────────────────────────────────────────┐
│           温度监控流程                           │
└──────────────────────────────────────────────────┘

                    ┌─────────────────┐
                    │  读取温度传感器 │
                    │ Task_ReadTemp() │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │ g_temperature   │
                    │    已更新       │
                    └────────┬────────┘
                             │
        ┌────────────────────┼────────────────────┐
        │                    │                    │
        ▼                    ▼                    ▼
   [< 60°C]            [60-70°C]            [≥ 70°C]
        │                    │                    │
        │               ┌────▼────┐              │
        │               │ 记录警告 │              │
        │               │ 日志消息 │              │
        │               └─────────┘              │
        │                    │              ┌────▼─────────┐
        │                    │              │ 立即关闭输出 │
        │                    │              │ DAC = 0      │
        │                    │              │ 发送错误日志 │
        │                    │              │ 中断工作     │
        │                    │              └─────────────┘
        │                    │
        ▼                    ▼
    ┌──────────┐        ┌──────────┐
    │ 继续工作 │        │ 继续工作 │
    │ 正常状态 │        │ 受限状态 │
    └──────────┘        └──────────┘

配置常量 (config.h):
  CFG_TEMP_PROTECT_THRESHOLD   = 70.0°C  (保护：关闭)
  CFG_TEMP_WARN_THRESHOLD      = 60.0°C  (警告：记录日志)
  CFG_TEMP_SAMPLE_INTERVAL_MS  = 1000    (1秒采样一次)
```

**温度传感器集成**：
```
┌─────────────────────────────────────────────┐
│         PCB 上的温度监控电路                 │
├─────────────────────────────────────────────┤
│                                             │
│  ┌────────────────┐       ┌─────────────┐ │
│  │ 温度传感器     │───────│ ADC Channel│ │

│  └────────────────┘       └──────┬─────┘ │
│                                  │        │
│                          ┌───────▼─────┐  │
│                          │  DMA/ADC    │  │
│                          │  采样转换   │  │
│                          └───────┬─────┘  │
│                                  │        │
│                          ┌───────▼──────────┐
│                          │Get_Kelvin_Temp() │
│                          │(库函数)          │
│                          └───────┬──────────┘
│                                  │
│                          ┌───────▼──────┐
│                          │ g_temperature│
│                          │ (全局变量)   │
│                          └───────────────┘
└─────────────────────────────────────────────┘
```

**代码流程示例**：
```c
void Task_MonitorProtection(void *argument)
{
    for (;;) {
        // 1. 读取温度
        Task_ReadTemperature(&g_temperature);
        
        // 示例：读到 65°C
        
        // 2. 检查是否超过保护阈值 (70°C)
        if (Task_CheckTemperatureProtection() != 0) {
            // 温度 >= 70°C
            LOG_ERROR(LOG_MOD_TEMP, "Temperature protection triggered! (65.0°C)");
            
            // 立即关闭输出
            Task_ApplyCurrentSetting(0);  // 设置电流为 0mA
            
            // 保护生效，等待手动复位
        }
        else if (g_temperature >= CFG_TEMP_WARN_THRESHOLD) {
            // 温度 >= 60°C 但 < 70°C
            LOG_WARN(LOG_MOD_TEMP, "Temperature warning: 65.0°C");
            // 继续工作，但记录警告
        }
        
        osDelay(1000);  // 每秒检查一次
    }
}
```

**实际应用场景**：
- 电子负载工作时产生热量
- 每秒检查一次芯片温度
- 如果温度达到 70°C，立即断电保护硬件
- 用户可从日志看到温度变化趋势

## 总结

这次优化的核心理念是：**关注点分离（Separation of Concerns）**

✅ **FreeRTOS 层** - 只负责任务调度  
✅ **业务逻辑层** - 实现具体功能  
✅ **硬件驱动层** - 与芯片交互  
✅ **配置层** - 统一参数管理  
✅ **日志层** - 统一调试输出  

**结果**：
- 代码更易理解
- 问题更易定位
- 功能更易添加
- 项目更易维护
- 团队合作更高效
